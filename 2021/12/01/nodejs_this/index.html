1. Global Object

nodejs 中有一個全域的 Object, 稱為 Global Object, 可以在任何地方被訪問不需要特別被引用, 也就是全域變數，例如直接輸出 __filename 可顯示目前正在執行的腳本文件名

console.log(__filename); // 輸出 D:\project\test.js

2. this

這裡探討的是 nodejs 的 this, 如果是在非嚴格模式下, 可以發現在函式內的 this 其實就是指向 global object, 如果是嚴格模式下, 則不允許函式內定義不存在的屬性。

/* 當 this 在函式內 */
function test(){
  this.name = "Jeremy"
}
test();
console.log(this)        // 輸出 {} 空物件
console.log(this.name)   // 輸出 undefined
console.log(globa.name)  // 輸出 Jeremy
console.log(name)        // 輸出 Jeremy

而如果是在全域的範圍內使用, 會發現是指向 module.exports

/* 當 this 在 global scope 中 */
this.name = "Jeremy";
console.log(module.exports.name); // 輸出 Jeremy

最後很直覺的, 如果 this 在建構函式或是類別建構子中的話,就會成為其實例的元件。

/* 當 this 在 class 中 */
class Person {
  constructor(name, age){
    this.name = name;
    this.age = age;
  }
  showName(){
    console.log(this.name)
  }
}
const person = new Person("Jeremy", 33); 
person.showName()        // 輸出 Jeremy
console.log(this.name)   // 輸出 undefined
